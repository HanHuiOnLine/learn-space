第0章: 前言
#####0.1基本准则
[建议]坚持一致性原则
在项目中维护别人的代码时,应该遵守一致性原则,即修改后的代码的编程风格应该与之前的代码保持一致.
#####0.2使用必读
不同的要求,严格程度不同,参照 RFC2119 分成三个等级:
1.[强制]  表示必须遵守(MUST).
2.[建议]  表示一般情况下应该遵守的(SHOULD),在某些特定的情况下可以忽略这个要求.
3.[可选]  表示这个要求可以根据个人喜好,自由选择.
